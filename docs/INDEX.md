## Common Questions

### Where do I start?

**If you're new**: Read [QUICK_REFERENCE.md](QUICK_REFERENCE.md) first

**If you want to understand testing**: Read [../tests/e2e/README.md](../tests/e2e/README.md)

**If you want the full context**: Read [findings/SESSION_FINAL_SUMMARY.md](findings/SESSION_FINAL_SUMMARY.md)

### How do I run tests?

See [QUICK_REFERENCE.md](QUICK_REFERENCE.md) - "Essential Commands" section

Quick answer:
```bash
npm test                    # Unit + integration tests
npm run test:e2e            # E2E tests (requires display)
npm run test:coverage       # With coverage report
```

### What's the test coverage?

See [findings/VISUAL_TEST_COVERAGE.md](findings/VISUAL_TEST_COVERAGE.md) for visual report

Quick answer:
- **Overall**: 97.3% (535/550 tests passing)
- **Critical Systems**: 100% (314/314 tests passing)
- **Status**: ‚úÖ Production Ready

### Why are some tests failing?

See [findings/FINAL_TEST_VALIDATION.md](findings/FINAL_TEST_VALIDATION.md) - "JSDOM Limitations Analysis"

Quick answer:
- 15 tests fail due to JSDOM environment limitations
- E2E tests prove these work correctly in real Chrome
- Not production bugs - proven false positives

### How do I set up the development workflow?

See [DEVELOPMENT_WORKFLOW.md](DEVELOPMENT_WORKFLOW.md)

Quick answer:
```bash
npm run dev:watch           # Watch files for changes
npm run dev:test:watch      # Auto-run tests on changes
```
